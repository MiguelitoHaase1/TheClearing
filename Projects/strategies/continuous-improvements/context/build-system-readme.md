# Build System — Product Strategy & Planning for Software Agents

Enable anyone who wants to build a piece of software to go through a journey where they build a proper product strategy that's outcome-driven and translates into ongoing evals and continuous improvements as new ideas come up. An ongoing project — much like Karpathy's AutoResearch — that optimizes day after day based on both its own algorithmic improvement and user feedback.

This system is split into 7 stages across the product lifecycle, each in its own subfolder:

1. **product-strategy/** — Overarching outcome contexts, vision, and jobs to be done - product agnostic. Solves problems in the real world, that will also exist two decades from now.
2. **archetype-selection/** — Translate strategy into archetypical solutions and prototypes. This is where boundary conditions come to life
3. **eval-creation/** — Create evaluation metrics (0-1 scale) for continuous measurement, by starting with leading indicators of success.
4. **planning-beads/** — Long-term planning implemented for persistent execution on long-term sessions(using beads). Make sure to be evals-centric
5. **build-v1/** — Build the first version in a Ralph Wiggum loop, optimizing evals undeer all the conditions above
6. **add-ideas/** — Capture ideas and hypotheses on-the-go for continuous improvement of V1
7. **continuous-improvements/** — Nightly experiments and improvements driven by evals and new ideas/hypotheses
